# local-video

Скачивай видео с YouTube (и других источников, поддерживаемых yt-dlp) и получай к нему
субтитры и краткое содержание — в том числе **на русском**, даже если у видео нет
готовых субтитров. Всё работает **локально**: транскрипция через faster-whisper,
перевод и саммари — через локальную модель в Ollama. Никаких облачных API.

Цель — иметь видео всегда под рукой и всегда понятным на родном языке.

## Из чего состоит

```
local-video/
├── .claude/skills/local-video/   # Claude Code Skill — пайплайн обработки
│   ├── SKILL.md                  #   инструкция для Claude (когда что делать)
│   └── scripts/                  #   утилиты: probe, download_video, fetch_subs,
│                                 #   transcribe, translate, summarize, subs
├── player/                       # локальный веб-плеер (Flask): видео + кликабельный
│                                 #   транскрипт; см. player/README.md
└── models/, <библиотека видео>   # генерируемое/большое — не в git
```

**Скилл** — основное. Это не самостоятельная программа, а навык для
[Claude Code](https://claude.com/claude-code): оркестратор — сам Claude, а скрипты
выполняют детерминированные шаги и возвращают JSON. Работа интерактивная — при
неоднозначности (много языков субтитров, нет субтитров, длинное видео) Claude
спрашивает пользователя.

**Плеер** — отдельное веб-приложение для просмотра результата (видео рядом с
транскриптом, перемотка по клику, поиск). Подробности — в `player/README.md`.

## Что делает пайплайн

```
ссылка → метаданные → скачать видео
       → субтитры: готовые (.srt/.vtt) ИЛИ транскрипция аудио (faster-whisper)
       → если не русский: перевод на русский (Ollama)
       → саммари по каждому транскрипту на его языке (Ollama)
```

На выходе для каждого видео — своя подпапка:

```
<название видео>/
├── <название>.mp4
├── <название>.<lang>.vtt          # субтитры (оригинал)
├── <название>.<lang>.summary.md   # саммари (на языке оригинала)
├── <название>.ru.vtt              # перевод на русский
└── <название>.ru.summary.md       # саммари на русском
```

Субтитры на выходе всегда в формате `.vtt`; саммари — в Markdown.

## Требования

- **Python 3.10+** и зависимости: `pip install -r .claude/skills/local-video/requirements.txt`
- **ffmpeg** (системно): `brew install ffmpeg` / `apt install ffmpeg`
- **Ollama** с моделью для перевода и саммари, например:
  `ollama pull gemma2` (в коде по умолчанию используется тег модели из конфига)
- **Модель faster-whisper** для транскрипции (если у видео нет готовых субтитров) —
  скачивается отдельно в `models/` (см. ниже).

## Установка

1. Клонируй репозиторий.
2. Поставь зависимости Python, ffmpeg и Ollama (см. «Требования»).
3. Запусти `./install.sh` — он создаст личный `config.json` из шаблона, сделает скилл
   доступным из любой папки (симлинк в `~/.claude/skills/`) и проверит окружение.
4. Отредактируй `config.json` под себя (папка-библиотека, модель Whisper и путь к ней,
   модель Ollama, хост).
5. Скачай модель faster-whisper (например, `small` или `medium`) в папку из
   `whisper_model_dir`. Если прямая загрузка с Hugging Face медленная, можно взять ту же
   модель в формате CTranslate2 с зеркала ModelScope (`pengzhendong/faster-whisper-<size>`).

## Конфигурация

Настройки — в `config.json` (создаётся из `config.example.json`, в git не попадает):

| Поле | Назначение |
|---|---|
| `library` | папка-библиотека, куда складываются все видео (по подпапке на видео) |
| `whisper_model` | размер модели Whisper (`tiny`/`base`/`small`/`medium`/…) |
| `whisper_model_dir` | путь к локальной CTranslate2-модели faster-whisper |
| `ollama_model` | модель Ollama для перевода и саммари |
| `ollama_host` | адрес Ollama |

## Использование

Открой проект в Claude Code и попроси обычными словами, например:

> скачай видео https://www.youtube.com/watch?v=… и сделай русские субтитры

Claude пройдёт весь пайплайн, при необходимости уточнит детали и сложит результат в
подпапку с названием видео. Затем результат можно открыть в плеере (`player/`).
