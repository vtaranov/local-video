# local-video — проектные решения

**Цель:** иметь видео всегда под рукой и всегда понятное на родном языке.

**Форма продукта:** не автономное приложение, а **Claude Code Skill**, который Claude использует интерактивно по поручению пользователя. Оркестратор — Claude; скрипты выполняют только детерминированные операции.

**Размещение скилла:** локально в проекте — `local-video/.claude/skills/local-video/`.

## Стек

- `yt-dlp` — скачивание видео и субтитров
- `ffmpeg` — извлечение аудио
- `faster-whisper` (локально, модель `medium` по умолчанию) — транскрипция, когда субтитров нет
- **Ollama + `gemma4:latest`** — перевод субтитров на русский (`qwen2.5:3b` как быстрый запасной)
- Claude API **не используется** — всё локально

## Форматы субтитров

- Вход: парсим `.srt` **и** `.vtt`
- Выход: все генерируемые субтитры — только `.vtt`

## Поведение

- Видео — качать **всегда** (флага отключения нет)
- Выходная папка — задаётся аргументом
- Hardsub **не** делаем — только отдельные `.vtt` рядом с видео

## Архитектура скилла

```
local-video/.claude/skills/local-video/
├── SKILL.md                    # инструкция / дерево решений для Claude
├── scripts/
│   ├── probe.py                # метаданные + список дорожек субтитров (JSON)
│   ├── download_video.py       # скачать видео (yt-dlp)
│   ├── fetch_subs.py           # скачать выбранные дорожки субтитров
│   ├── transcribe.py           # faster-whisper: аудио → .vtt
│   ├── translate.py            # Ollama gemma4: .vtt → русский .vtt
│   ├── summarize.py            # Ollama: транскрипт → саммари .md (на его языке)
│   └── subs.py                 # общий модуль: парсинг srt/vtt + детект языка
├── requirements.txt
└── README.md
```

Скрипты возвращают машиночитаемый JSON в stdout и внятные коды ошибок; решений не принимают.

## Логика пайплайна

```
probe → решение/вопрос пользователю → download_video
      → (fetch_subs ИЛИ transcribe) → если не русский: translate
      → summarize по КАЖДОМУ транскрипту (на его языке) → итог
```

## Саммари

- По **каждому** транскрипту (оригинал и перевод) делается саммари на его языке.
- Имя файла: `<name>.<lang>.summary.md` (Markdown, рядом с видео и `.vtt`).
- Модель — та же, что для перевода (по умолчанию `gemma4:latest`).
- Структура: краткое содержание, ключевые тезисы, структура по таймкодам.
- Для длинных транскриптов — map-reduce (поблочно + финальный свод).

## Интерактив (обсуждается с пользователем)

- Много дорожек субтитров на разных языках → какие нужны
- Авто- vs ручные субтитры (авто YouTube часто с ошибками)
- Нет субтитров → подтвердить транскрипцию (дольше)
- Длинное видео → предупредить о времени
- Неуверенный детект языка
- Ошибки скачивания / приватное / гео-блок
- Отсутствие yt-dlp / ffmpeg / моделей

## Плеер транскриптов

**Что это:** четвёртая стадия продукта — «смотреть». Локальный веб-плеер, который
потребляет результат скилла (видео + `.vtt` из `output/`) и показывает видео рядом
с кликабельной лентой транскрипта.

**Тип артефакта:** пользовательское веб-приложение, а **не** скилл. Поэтому лежит
отдельно — в `player/`, а не в `.claude/skills/`. Запускается вручную.

**Стек:** Flask (бэкенд: API + раздача файлов с HTTP Range для перемотки) +
ванильный JS/CSS (фронтенд: парсинг VTT, транскрипт, поиск, подсветка). Без сборки,
без фреймворков. Окружение — `player/venv/`.

**Возможности:** клик по реплике → перемотка; выбор языка из имён `.vtt`;
автоскролл с кнопкой вкл/выкл и подсветкой текущей реплики; поиск по тексту;
смена папки прямо из интерфейса; авто-подбор порта при занятом 5000.

**Размещение:**

```
local-video/
├── .claude/skills/local-video/   # скилл: download → subs → translate
├── output/                        # результат скилла (видео + .vtt)
└── player/                        # плеер (читает ../output по умолчанию)
    ├── app.py  templates/  static/  venv/
    └── README.md                  # запуск и детали
```

**Запуск:** `cd player && ./venv/bin/python app.py` → открыть напечатанный адрес.
Подробности — в `player/README.md`.

**Решено не делать:** скилл-обёртку для запуска плеера через Claude (на старте
достаточно README; вернуться, если запуск станет частым).
