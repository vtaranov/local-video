# local-video

Скачивание видео + получение субтитров на русском. **Оркестратор — ты (Claude).**
Скрипты делают детерминированную работу и возвращают JSON в stdout; решения и
диалог с пользователем — на тебе.

## Конфигурация (прочитай первым делом)

Все настройки — в конфиге. Получи их так (из каталога `scripts/`):
```
python config.py
```
→ JSON с полями: `library` (папка-библиотека для всех видео),
`whisper_model_dir` (путь к модели транскрипции), `ollama_model`, `ollama_host`.
Скрипты `transcribe.py`/`translate.py`/`summarize.py` берут эти значения как
дефолты автоматически — переопределяй только по необходимости.

## Принципы

- Видео качается **всегда**.
- **Каждое видео — в свою подпапку.** База вывода по умолчанию = `library` из конфига.
  **При старте сообщи пользователю дефолтную папку (`library`) и спроси, не сменить ли её.**
  Имя подпапки = `<очищенный заголовок> [<id>]` — не считай его вручную, бери из
  `library.py folder` (см. шаг 2.5). Дедупликация по `id` тоже там.
- **Метаданные источника и дедуп.** В каждой папке видео — сопроводительный файл
  `.source.json` (id, url, title, extractor, дата), пишется через `library.py sidecar`.
  Повторная обработка того же видео определяется по `id` (через `library.py`), не по
  совпадению заголовков.
- На выходе субтитры всегда в **.vtt** (и оригинал, и перевод).
- Hardsub не делаем — только отдельные файлы рядом с видео.
- Перевод — локально через Ollama (модель из конфига). Никаких облачных API.
- При любой неоднозначности — **спрашивай**, не угадывай.

## Окружение (проверь при первом запуске)

Скрипты лежат в `scripts/`. Запускай их рабочей директорией `scripts/`
(там лежит общий модуль `subs.py`), python — текущий из окружения пользователя.

Нужны: `yt-dlp`, `faster-whisper`, `langdetect` (pip, см. `requirements.txt`),
`ffmpeg` (системно, для merge видео и декода аудио), Ollama с моделью `gemma4:latest`.
Если чего-то нет — скажи пользователю, что именно поставить, не пытайся скрыто чинить.

## Рабочий процесс

### 1. Разведка
```
python probe.py "<URL>"
```
Получишь JSON: `id`, `extractor`, `title`, `duration`, `manual_subs[]`, `auto_subs[]`
(каждая дорожка — `{lang, name, exts}`).

Покажи пользователю кратко: название, длительность, какие субтитры доступны.

### 2. Реши, откуда брать субтитры (дерево решений)

- **Есть ручные субтитры, одна дорожка, русская** → берём её, перевод не нужен.
- **Есть ручные, одна дорожка, не русская** → берём её, дальше перевод.
- **Много дорожек на разных языках** → ПОКАЖИ список и СПРОСИ, какие нужны
  и нужен ли перевод каждой на русский.
- **Только авто-субтитры (`auto_subs`)** → предупреди, что авто-субтитры YouTube
  бывают с ошибками, и СПРОСИ: брать их (`fetch_subs --auto`) или
  транскрибировать звук заново через Whisper (точнее, но дольше).
- **Субтитров нет вообще** → предупреди, что будет транскрипция (может быть долго
  для длинного видео), подтверди и используй `transcribe.py`.

Если `duration` большой (> ~30 мин) и предстоит транскрипция или перевод —
предупреди о времени до запуска.

### 2.5. Определи папку и проверь дедуп
```
python library.py folder --library "<LIBRARY>" --title "<title>" --id "<id>"
```
→ `{dir, exists}`. `<LIBRARY>` — папка из конфига (или заданная пользователем).
- `exists: true` → видео уже обработано. Сообщи пользователю и **спроси**, нужно ли
  перекачать/обновить; по умолчанию пропусти.
- `exists: false` → `dir` — целевая папка `<LIBRARY>/<title> [<id>]/`. Все файлы — туда.

### 3. Скачай видео
Все дальнейшие файлы (видео, `.vtt`, `.md`) — в `dir` из шага 2.5 (обозначим `<DIR>`).
```
python download_video.py "<URL>" --out "<DIR>"
```
→ `video_path`.

### 4. Получи исходные субтитры

Готовые дорожки:
```
python fetch_subs.py "<URL>" --langs en,es --out "<DIR>" [--auto]
```
Для авто-субтитров (`--auto`) скрипт сам тянет формат `json3` (не `vtt`!) и
конвертирует в чистый `.vtt` через `subs.parse_json3` — родной vtt-экспорт
YouTube для авто-субтитров эмулирует прокрутку перекрывающимися репликами
(см. примечание в «Обработка ошибок» ниже). Ничего дополнительно делать не нужно.

Либо транскрипция (нет субтитров / пользователь выбрал Whisper):
```
python transcribe.py "<VIDEO>" --out "<DIR>/<name>.<lang>.vtt" [--lang en]
```

### 5. Определи язык и переведи при необходимости

Если язык дорожки неизвестен — определи по метаданным или попроси `subs.py`
(`detect_language`). Если **не русский**:
```
python translate.py "<IN.vtt>" --out "<DIR>/<name>.ru.vtt" --model gemma4:latest
```
Тайминги сохраняются на уровне батча (начало/конец окна совпадают с
оригиналом), но число и границы реплик внутри батча могут отличаться от
исходных — модели разрешено объединять фрагменты предложений при переводе
(см. docstring `translate.py`). Для русского оригинала перевод пропусти.

### 6. Саммари по всем транскриптам
Для **каждого** полученного транскрипта (и оригинал, и перевод) сделай саммари
**на языке этого транскрипта** той же моделью, что использовалась для перевода
(по умолчанию `gemma4:latest`):
```
python summarize.py "<IN.vtt>" --out "<DIR>/<name>.<lang>.summary.md" \
       --model gemma4:latest --lang <lang> --title "<название видео>"
```
То есть для `<name>.en.vtt` → `<name>.en.summary.md` (англ.),
для `<name>.ru.vtt` → `<name>.ru.summary.md` (рус.). Если транскрипт только один —
саммари одно. Модель для саммари = модель перевода.

### 7. Запиши метаданные источника (сопроводительный файл)
```
python library.py sidecar --dir "<DIR>" --id "<id>" --url "<URL>" \
       --title "<title>" --extractor "<extractor>"
```
Создаёт `<DIR>/.source.json` — по нему работает дедупликация при следующих запусках.

### 8. Итог
Короткое резюме: что скачано, какие файлы получены и где лежат
(`<name>.<lang>.vtt` — субтитры, `<name>.<lang>.summary.md` — саммари). Без лишних слов.

## Просмотр в плеере

В проекте есть локальный веб-плеер (отдельный компонент в `player/`): видео рядом с
кликабельным транскриптом и саммари + экран библиотеки. Запускай его, когда
пользователь просит **показать/открыть/запустить видео в плеере** («…и покажи в
плеере»); в конце пайплайна можно предложить.

1. Узнай, где плеер и чем его запускать:
   ```
   python play.py
   ```
   → `{player_dir, app, python, flask_ok, ready}`. Если `ready: false` — скажи
   пользователю выполнить `install.sh` (он настроит плеер) и не запускай.
2. Подними сервер **в фоне** интерпретатором `python` из вывода:
   - конкретное видео: `"<python>" "<app>" --dir "<папка видео из шага 2.5>"`
   - список библиотеки (без `--dir`): `"<python>" "<app>"`
3. Прочитай из вывода сервера строку `http://127.0.0.1:<порт>` и дай её пользователю
   (на macOS можно `open <url>`). Порт выбирается автоматически; библиотеку плеер
   берёт из конфига сам.

## Именование файлов

Все файлы клади рядом с видео, имя = имя видео + суффикс:
- субтитры: `<name>.<lang>.vtt` (оригинал и перевод `<name>.ru.vtt`) — плеер
  (VLC/MPV) подхватит их автоматически;
- саммари: `<name>.<lang>.summary.md` — Markdown на языке соответствующего транскрипта.

## Пакетный режим (плейлист / канал)

Триггер: «скачай все новые видео из плейлиста `<url>`» и подобное.

1. Перечисли видео:
   ```
   python list_playlist.py "<url>"
   ```
   → `{playlist, count, entries:[{id,title,url}]}`.
2. Узнай уже скачанное:
   ```
   python library.py index --library "<LIBRARY>"
   ```
   → список `{id,…}`.
3. **Недостающие** = id из плейлиста, которых нет в индексе. Сообщи пользователю:
   всего в плейлисте N, уже есть M, новых K. Назови дефолтную `<LIBRARY>` и спроси,
   не сменить ли. Если новых много или видео длинные — подтверди объём.
4. Обработай недостающие **последовательно, одним субагентом**: запусти субагент
   со скиллом `local-video`, передав ему список недостающих `url`, путь `<LIBRARY>`
   и **политику ниже**. Субагент идёт по списку по очереди, без вопросов по каждому
   видео; на каждое выполняет обычный пайплайн (шаги 2.5–7).
5. Итог: что добавлено, что пропущено (уже было / ошибки) — таблицей.

### Политика пакетной обработки (без интерактива)

В пакете НЕ задавай вопросов по каждому видео — действуй по правилам:
- **Папка:** одна `<LIBRARY>` на весь прогон; каждое видео — в свою подпапку (шаг 2.5).
- **Субтитры:** ручные (предпочесть оригинальный язык) → если нет, авто (`fetch_subs --auto`)
  → если нет, транскрипция Whisper (модель из конфига).
- **Языки:** оригинал + перевод на русский (если оригинал не русский).
- **Саммари:** по всем транскриптам. **Сайдкар:** записать для каждого видео.
- **Пропускай и фиксируй в отчёте** (не блокируйся, не спрашивай): приватное/гео-блок,
  ошибка скачивания, видео без субтитров И длиннее ~1 часа (дорогая транскрипция).

## Обработка ошибок

Скрипт при ошибке печатает `{"error": "..."}` и возвращает код 1.
Не глотай ошибку — покажи пользователю текст и предложи вариант:
приватное/гео-блок видео, недоступный Ollama, отсутствие ffmpeg и т.п.

**Ложная «video is not available».** Если `probe.py`/`download_video.py`/`fetch_subs.py`
сообщают `This video is not available`, а видео при этом открывается в браузере —
это известный баг текущей версии yt-dlp с дефолтным web-клиентом (проблема с
расшифровкой плеера), а не реальная блокировка. Проверь: повтори с
`--extractor-args "youtube:player_client=android"` (поддерживается всеми тремя
скриптами, можно указывать несколько раз). Если заработало — используй этот
флаг на всех шагах для этого видео.

**Перекрывающиеся реплики в авто-субтитрах YouTube.** Если видишь, что соседние
реплики транскрипта почти дублируют друг друга («I'm excited to be here» / «I'm
excited to be here and to kick us off.» / «off.» / ...) — это не баг распознавания
речи и не наша транскрипция, а артефакт `.vtt`-экспорта YouTube для авто-субтитров:
он эмулирует побуквенную/пословную прокрутку, поэтому соседние vtt-реплики
показывают накопленный текст с перекрытием. В родном формате `json3` того же
запроса перекрытия текста нет (только перекрываются тайминги показа, что тоже
обработано). Поэтому `fetch_subs.py --auto` качает именно `json3` и сам
конвертирует в чистый `.vtt` (см. шаг 4) — если где-то всё же встретится vtt с
такими дублями (например, старый файл скачан до этого фикса), просто перекачай
дорожку заново через текущий `fetch_subs.py`.
