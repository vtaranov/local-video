# Плеер транскриптов

Локальный веб-плеер: **экран библиотеки** со списком всех видео + просмотр выбранного
(видео и кликабельная лента транскрипта рядом). Потребляет результат скилла `local-video`.

## Запуск

```bash
cd player
./venv/bin/python app.py
```

В консоли появится адрес, например `http://127.0.0.1:5000` — открыть в браузере.
Остановить — `Ctrl+C`.

Порт подбирается автоматически: если 5000 занят (на macOS его держит AirPlay Receiver),
сервер уйдёт на 5001 и т.д. и напечатает реальный адрес.

## Экран библиотеки

При старте плеер показывает **список видео из библиотеки** — каждое в своей подпапке
`<название> [id]/` (название берётся из `.source.json`, если есть). Клик по карточке
открывает видео; кнопка «← Библиотека» возвращает к списку. Путь библиотеки можно
сменить полем в шапке списка.

Папка-библиотека по умолчанию берётся из конфига скилла
(`../.claude/skills/local-video/config.json`, поле `library`); если конфига нет —
`~/all-local-videos`.

## Аргументы

```bash
./venv/bin/python app.py --library "/путь/к/библиотеке"  # корень со списком видео
./venv/bin/python app.py --dir "/путь/к/одной/папке"     # сразу открыть одно видео
./venv/bin/python app.py --port 8080                      # стартовый порт
LIBRARY="/путь" ./venv/bin/python app.py                  # библиотека через env
```

## Что нужно в папке

Рядом должны лежать:

- видео — `.mp4` / `.webm` / `.mkv` / `.mov` / `.m4v` (берётся первое по алфавиту);
- субтитры `.vtt`, по одному на язык. Язык определяется по концу имени файла:
  `…en.vtt` → English, `…ru.vtt` → Русский.

## Возможности интерфейса

- **Транскрипт справа** — клик по реплике перематывает видео на этот момент.
- **Выбор языка** — выпадающий список, собран из имён `.vtt`.
- **Автоскролл** — кнопка вкл/выкл; панель следует за видео и подсвечивает текущую реплику.
- **Поиск** — фильтрует реплики и подсвечивает совпадения со счётчиком.
- **Библиотека** — стартовый экран со списком видео; клик открывает, «← Библиотека»
  возвращает. Путь библиотеки меняется полем в шапке списка.

## Архитектура

```
player/
├── app.py              # Flask: /api/library, /api/media, /api/open, /api/home,
│                        #        /media/<file> (с HTTP Range)
├── templates/index.html
├── static/
│   ├── app.js          # парсер VTT, рендер транскрипта, поиск, подсветка
│   └── style.css
└── venv/               # окружение с Flask
```

- Бэкенд хранит реальные пути; браузер работает только с URL `/media/...` и `/api/...`.
- Видео отдаётся с поддержкой HTTP Range (`send_file(conditional=True)`) — перемотка
  по большому файлу мгновенная.
- VTT парсится на клиенте: склеивает подряд идущие дубли (артефакт авто-субтитров),
  убирает inline-теги.
```
